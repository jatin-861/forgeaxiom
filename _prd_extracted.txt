feTurbulence + feDisplacementMap filters combined with CSS hue-rotation animation, creating an organic,
living-glass appearance. Apple's "liquid glass" (iOS 26) style — but darker, more cosmic.

3D CRYSTAL IMPLEMENTATION
6.1 AI Generation Pipeline — How to Create the Crystal
STEP 1 — Text-to-3D
Go to meshy.ai → Text to 3D → Prompt: "Dark obsidian crystal gem, faceted, sharp
geometric faces, translucent purple edges, octahedral elongated form, physically-based
rendering material, PBR" → Generate → Download .glb
STEP 2 — Alternative
Use spline.design → Open AI menu → "Generate 3D object" with same prompt → Export
as .splinecode for direct Spline React embed
STEP 3 — Refinement
Import .glb to Blender → Add Emission shader on edges (purple, strength 2.0) → Add
interior volumetric scatter (purple, density 0.4) → Re-export .glb optimized
STEP 4 — Compression
Run through gltf.report + gltfpack CLI: gltfpack -i crystal.glb -o crystal_opt.glb -cc →
Target < 2MB
STEP 5 — Fallback
Create CSS-only 3D crystal using clip-path polygon for browsers/devices that can't
handle WebGL (< 5% of modern browsers)

6.3 Mouse Parallax System
Hook
Custom useMouseParallax() hook — normalizes cursor position to [-1, 1] range relative to
viewport center
Smoothing
useSprings with config: { mass: 1, tension: 170, friction: 26 } — gives organic spring feel
Crystal Response
X-axis mouse → crystal drifts ±0.8 units · Y-axis → crystal tilts ±15deg
Particle Response
Mouse enter WebGL canvas → particles repel within 80px radius using distance
calculation
Text Parallax
Hero text layers move at different speeds on scroll: headline 0.3x / subtext 0.5x / bg 0.1x
Performance
All parallax calculations happen in useFrame (R3F) or rAF — never in React state
Mobile
Replaces mouse events with DeviceOrientationEvent (gyroscope) for parallax effect on
mobile

10.1 Mobile-First Philosophy
Since the target includes phone users, every section is designed mobile-first. The 3D crystal is displayed on
desktop and mid-range phones (WebGL check). On older/low-end phones, a beautiful CSS-only version replaces
it. All glassmorphism effects are performance-tested on mobile Chrome.
Breakpoints
xs: 375px / sm: 640px / md: 768px / lg: 1024px / xl: 1280px / 2xl: 1536px
Crystal on Mobile
Smaller canvas (300×300), particle count reduced to 400, no shadow casting
Glassmorphism Mobile
backdrop-filter: blur(12px) — reduced blur for performance
Font Sizes
CSS clamp() — fluid scaling, no fixed breakpoint jumps
Navigation
Full-screen overlay menu, touch-friendly 48px tap targets minimum
Touch Gestures

Horizontal Scroll
